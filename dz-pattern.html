Help

Getting Started

Tap or click anywhere on the map to set a landing target. The app fetches winds aloft for that location and draws a suggested landing pattern. Tap or click again to move the target.

Tap or click 🔍 to open search. Type a drop zone name to find it in the USPA database, or type any city, address, or landmark to navigate the map there — then tap or click the map to place your landing target. Tap the crosshair button () in the search bar to jump the map to your current GPS location and close the search bar.

Final Heading

The heading bar at the bottom controls the final approach direction. By default it snaps to the surface wind direction. Drag the slider or tap/click the number field to type a heading directly. The ▲ marks the into-wind heading — tap or click it to snap back. Toggle L/R hand pattern with the L/R buttons.

Pattern Legs

The standard pattern has three legs — Downwind (orange), Base (teal), Final (yellow) — each with an independent Crab or Drift setting. Additional legs above the downwind can be added with the + Add Leg button in the Pattern Legs header — see Additional Legs below.

Crab: the app solves for a steered heading that produces the target ground track, accounting for wind. A dashed line (colored to match the leg) shows the steered heading direction when it differs meaningfully from the ground track.

Drift: the canopy is steered on a nominal heading and the wind carries it toward the turn point. A dashed line (colored to match the leg) shows the steered heading direction when wind drift is significant.

Defaults: Downwind = Crab, Base = Drift, Final = Crab.

Z Pattern: a checkbox on the Downwind leg (only shown when no extra legs are present). When enabled, the downwind leg flies in the same direction as the final approach leg — rather than the opposite direction as in a normal pattern. This produces a Z-shaped pattern instead of the usual C shape, and is used when the opening point is downwind of the target so the jumper needs to fly toward the landing area before turning base. Z pattern is independent of the Crab/Drift setting.

Additional Legs

Tap + Add Leg (in the Pattern Legs header) to add a leg above the downwind — useful for high opening altitudes, long spot, or multi-leg approaches. Legs are numbered 4, 5, … and displayed highest-altitude first. Each extra leg has:

Alt (ft AGL) — the altitude at which this leg begins. Must be at least 100 ft above the leg below it; the bottom-most extra leg must be at least 100 ft above the Downwind entry altitude.

Approach Hdg — the nominal heading the canopy flies on this leg (0–359°). Defaults to the downwind direction at leg altitude (wind-blows-toward heading); falls back to a 90° turn from the leg below in calm winds. Crab mode solves a steered heading to achieve this ground track; Drift mode steers the nominal heading directly.

Crab / Drift — same as standard legs. Each extra leg is independent.

Custom performance — per-leg canopy glide/speed/sink, same as standard legs.

The × remove button only appears on the most recently added leg. Remove legs from the top down.

Canopy Performance

The Default Canopy Performance section (in Settings) sets glide ratio, horizontal speed, and vertical speed — enter any two and the third is calculated. Use still-air, no-wind values. Vertical speed equals sink rate. These values are available from FlySight, Deep and Steep Insight, and similar logging devices.

Each leg also has an optional Custom performance checkbox. When enabled, that leg uses its own glide/speed/sink values instead of the defaults — useful for modeling braked flight on downwind and base with a full-flight final, for example. Enabling it pre-fills from the current defaults.

Pattern Altitudes

Set entry, base turn, and final turn altitudes (ft AGL) in Settings. These define the length of each leg — the higher the entry altitude and the better the glide ratio, the longer the pattern. The landing target is always 0 ft AGL.

Altitudes must be strictly ordered with a minimum 100 ft gap between each leg: Enter > Base + 100, Base > Final + 100. Extra legs above the downwind must each be at least 100 ft apart. The app shows an error in the status bar if any altitude constraint is violated.

Safety Regions (Canopy)

Three amber rings around the pattern entry point show where a canopy pilot must be at each altitude milestone to reach the entry point under canopy. All rings shift in the wind drift direction to account for accumulated drift during descent. Region Buffer (%) in Settings shrinks all rings by that fraction as a safety margin.

The yellow ring marks the required opening altitude position — you must open within this ring to reach pattern entry under canopy.

Safety Regions (Opening & Exit)

The blue dashed ring shows where you must exit the aircraft so that freefall drift carries you into the opening ring. Freefall drift is integrated across altitude bands using actual winds aloft at each level, assuming purely vertical descent — results are not applicable to tracking, angle, or movement dives.

Jump Run

A suggested jump run is drawn through the exit region center, into the wind at exit altitude. The ▲ marks the into-wind heading — tap or click it to snap. The heading field is also directly editable.

Exit Alt / Opening Alt: exit altitude must be above opening altitude — the app shows an error if they conflict. These altitudes drive the freefall drift integration and the exit/opening safety rings.

Freefall Speed: vertical descent speed during freefall (mph). Horizontal drift from wind is integrated across altitude bands from exit to opening altitude.

Aircraft IAS: indicated airspeed of the jump aircraft (kts), used to compute jump run ground speed and exit separation timing.

Region Buffer (%): shrinks all safety rings by this fraction as a conservative margin.

Offset: perpendicular distance from the landing spot to the jump run line. Auto-calculated when blank (shown in muted color). Enter a value to override — positive is right of flight path looking forward. Delete the value to return to auto.

🟢/🔴 green/red light distances: how far before (🟢) and after (🔴) the aircraft is directly over the landing area when it crosses the exit circle boundary. Measured along the jump run.

Exit separation: suggested time between groups based on the exit separation distance setting and jump run ground speed.

Winds Aloft

Winds are fetched from Open-Meteo (GFS). Teal rows are real model data points at pressure levels or fixed heights; grey rows are interpolated. The surface wind comes from the 10m wind field. Winds are cached for 20 minutes — the app checks every minute and automatically fetches fresh data when the cache expires. Individual values in the wind table can be edited manually to test different wind scenarios.

The Forecast slider lets you view winds up to +12 hours ahead. The effective forecast time appears in the center of the top bar — muted for current hour, teal for a future hour. Moving to a new location resets the forecast to the current hour.

Pull down on the top bar to manually refresh wind data.

Display Options (👁)

Toggle individual map layers on and off. Airspace layers: Jump Run, Exit Region, Canopy Regions. Pattern label layers: Turn Altitudes (entry/base/final altitudes at turn points), Leg Distances & Wind (distance, wind component, and seconds per leg), Headings & Steer (track heading and steered heading per leg — the steered heading dashed line is colored to match its leg), Leg Arrows (directional chevrons). The steered heading threshold (°) in Settings controls when steered heading lines appear. Settings are saved between sessions.

Settings Persistence

All settings are automatically saved and restored on next visit. Settings that depend on wind or location — final heading and jump run heading — are not saved and re-derive from wind data on each visit. Use Reset All Settings & Clear Data at the bottom of Settings to wipe everything and start fresh, including your terms agreement.
